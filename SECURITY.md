# Security Policy

## Supported Versions

This project is pre-`1.0`. Security fixes will be applied to the latest release on a best-effort basis.

## Reporting a Vulnerability

Do not open a public GitHub issue for suspected vulnerabilities.

Report security issues to Jacob Bourne at `jacob.bourne@gmail.com` with:

- A description of the issue
- Steps to reproduce or a proof of concept
- The affected version or commit, if known
- Any suggested remediation or mitigation

I will acknowledge receipt as soon as practical, confirm whether the issue is reproducible, and coordinate a fix and disclosure plan.
