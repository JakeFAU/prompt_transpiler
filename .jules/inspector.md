## 2024-05-11 - LLMAdjudicator.evaluate handles JSON bool/list gracefully **Edge Case:** The LLM returns a valid JSON string that is a boolean (`"true"`) or a list (`"[]"`) instead of the expected dictionary payload. **Learning:** While inner methods like `_apply_pairwise_verdicts` would throw an `AttributeError` or `TypeError` because they assume `data` is a dictionary, the outer `evaluate` method catches all `Exception`s, logs the failure, and safely returns a fallback score of `0.0`. The system degrades safely instead of crashing the pipeline. **Action:** Continue testing boundary payloads from external APIs (like LLM outputs) with unexpected but technically valid JSON types (lists, integers, strings, nulls) to verify error boundaries remain secure.
