## 2024-05-01 - LLMAdjudicator malformed JSON data types **Edge Case:** Tested how LLMAdjudicator handles valid JSON where the expected string fields (like verdicts or confidences) are instead lists, dicts, integers, or booleans. **Learning:** The system gracefully degrades malformed types (e.g. lists, booleans, integers) by returning None for verdicts/confidences or skipping constraint entries, avoiding exceptions during evaluation. **Action:** Continue fuzzing LLM JSON responses with mismatched primitive types in addition to completely invalid JSON.
