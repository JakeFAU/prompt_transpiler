## 2025-02-28 - Cache LLM Provider factory for performance **Learning:** The application repeatedly instantiates `LLMProvider` implementations during processing (e.g. `Decompiler` and `LLMAdjudicator` creating new clients on every eval call). Instantiating heavy async clients like `AsyncOpenAI` repetitively carries CPU and network initialization overhead. **Action:** Added `@functools.lru_cache(maxsize=None)` to `get_llm_provider` to ensure a single instance is used per provider per run, and updated tests with an `autouse` cache-clearing fixture.
