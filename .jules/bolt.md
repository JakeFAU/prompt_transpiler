## 2024-05-11 - Cache LLM Provider Factory **Learning:** Async LLM clients (like `AsyncOpenAI` or `AsyncAnthropic`) carry heavy initialization overhead. Their instances, or the factory functions creating them, should be cached to prevent performance degradation during repetitive processing. When applying `@functools.cache` to globally accessed factory functions, an `autouse=True` fixture calling `<function_name>.cache_clear()` in `conftest.py` is necessary to ensure test isolation. **Action:** Always cache async client instantiations and factory functions while ensuring proper test isolation via cache clearing.
