## 2024-05-09 - Caching LLM Provider Factories **Learning:** Async LLM clients (like `AsyncOpenAI`) carry heavy initialization overhead (e.g. setting up HTTP connection pools). Creating new instances inside hot loops or frequent pipeline steps significantly degrades performance and wastes memory. **Action:** Apply `@functools.cache` to factory functions creating these clients, and ensure test isolation by clearing the cache in an `autouse=True` fixture in `conftest.py`.
