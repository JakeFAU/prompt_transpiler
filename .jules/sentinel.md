## 2024-05-20 - SQL Injection via dynamic columns **Vulnerability:** SQL injection in `DuckDBJobStore.update_job` and `SQLiteJobStore.update_job` due to using f-strings to dynamically format column names in `UPDATE` queries without restricting keys. **Learning:** Although values were parameterized, dynamically formatting keys is unsafe because `bandit` (and real attackers) see it as a vulnerability when arbitrary inputs can reach it. Using `frozenset` validations is a lightweight, secure way to solve this. **Prevention:** Validate all dynamically formatted SQL identifiers against a strict whitelist of known safe string literals before interpolating them into queries.
