## 2026-04-24 - [Store SQL Injection via F-Strings] **Vulnerability:** Unsanitized dictionary keys were being interpolated directly into DuckDB and SQLite `UPDATE` queries using f-strings in `src/prompt_transpiler/jobs/store.py` (`_to_update_clause`). **Learning:** While the input dictionary typically contained hardcoded keys from the application, the lack of explicit validation left the backend vulnerable to SQL injection if an upstream API route ever unpacked unsanitized user inputs. **Prevention:** Validate all dynamic query keys against a strictly defined module-level `frozenset` (`ALLOWED_UPDATE_COLUMNS`) and utilize Bandit's `# nosec B608` to explicitly document the intentional and now-safeguarded f-string usage.
