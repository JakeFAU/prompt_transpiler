## 2024-04-20 - [Fix B608 and B104] **Vulnerability:** SQL injection via string concatenation in UPDATE query and binding API to 0.0.0.0 **Learning:** B104 binds app to all interfaces, exposing server. B608 was caused by blindly trusting 'db_key' derived from an arbitrary 'fields' dict in python during an UPDATE statement without column whitelisting. **Prevention:** Set default HOST to 127.0.0.1 and always whitelist column names when they are dynamically generated for a query, even if the inputs aren't immediately attacker-controlled, to prevent future SQLi paths.
