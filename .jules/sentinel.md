## 2025-02-21 - Prevent SQL Injection in Job Stores **Vulnerability:** Possible SQL injection via unvalidated column names in dynamic UPDATE queries. **Learning:** Although dynamic UPDATE clauses are convenient, relying on implicit valid dictionary keys without explicit identifier validation leaves the execute call vulnerable to injection. Bandit correctly flags B608 for this pattern. **Prevention:** Always validate dynamic column names with .isidentifier() before inserting them into an f-string SQL query, apply this to all matching interface implementations uniformly, and suppress the resulting B608 with # nosec B608 once properly validated.
