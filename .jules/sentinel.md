## 2026-04-27 - [Dynamic SQL Column Injection] **Vulnerability:** Unsanitized dynamically constructed SQL queries (`UPDATE SET {columns} ...`) where keys were mapped directly from request dictionaries. **Learning:** While values were parameterized, dynamically built column names (keys in dicts) bypass driver sanitization and create severe SQLi risks if user input populates those keys. **Prevention:** Always validate that dynamically injected structure variables (like column names) adhere strictly to identifier format rules (e.g., using `str.isidentifier()`) before query interpolation.
