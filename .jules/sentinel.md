## 2024-05-13 - [Hardcoded Bind All Interfaces] **Vulnerability:** B104 (hardcoded_bind_all_interfaces) detected in app.py binding Flask server to 0.0.0.0. **Learning:** Although development servers are not usually exposed, binding to all interfaces defaults can accidentally expose development endpoints locally or internally depending on infrastructure. **Prevention:** Default to local loopback (127.0.0.1) for local runner bindings and rely on explicit environment configurations for production exposure.

## 2024-05-13 - [Bandit SQL Injection Warning B608] **Vulnerability:** Bandit flagged `f"UPDATE compile_jobs SET {columns} WHERE job_id = ?"` as a potential SQL injection risk (B608). **Learning:** When columns are dynamically generated for SQL queries via dictionary keys, even if the user data itself is parameterized (the `values`), the column keys could be vectors for SQL injection. To safely use dynamic column names in f-strings, they should be validated first, for example, using `.isidentifier()`. After proper validation, `# nosec B608` is needed to suppress the false positive in Bandit. **Prevention:** Always validate dynamic identifiers constructed in SQL statements or map them against a strictly allowed set of schema columns.
