## 2025-05-16 - Prevent Bandit False Positives on Validated Dynamic SQL **Vulnerability:** B608 false positive on dynamically generated SQL columns from dictionary keys. **Learning:** Although the keys were statically supplied by code, dynamically generating `f"UPDATE SET {columns}"` correctly flags B608. Adding `.isidentifier()` validation protects against rogue strings while `# nosec B608` suppresses the false positive. **Prevention:** Always validate dynamically generated column names with `.isidentifier()` and append `# nosec B608` to the execution line to cleanly suppress the linter while enforcing security.
