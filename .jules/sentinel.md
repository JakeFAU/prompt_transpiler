## 2026-04-25 - Prevent SQL Injection in Dynamic UPDATE Statements **Vulnerability:** Unvalidated dynamic column names used in f-string queries in `_to_update_clause` within `prompt_transpiler.jobs.store`. **Learning:** Standard SQL parameterization only protects values, not column identifiers. When updating dynamic fields, keys mapping to column names must be explicitly validated as safe. **Prevention:** Always validate dynamic column identifiers using `key.isidentifier()` or an allowlist before injecting them into a SQL string, even if the data source currently seems safe.
