## 2026-04-21 - [SQL Injection via Dynamic Columns] **Vulnerability:** Unsanitized column names passed dynamically to SQL UPDATE statements in `_to_update_clause`. **Learning:** Python dictionaries derived from kwargs (`**fields`) were directly unpacked into f-strings for SQL query construction. While values were parameterized, column names cannot be, making the system vulnerable if a caller could manipulate kwargs keys. **Prevention:** Always validate dynamic column names against an explicitly defined `ALLOWED_COLUMNS` allowlist (e.g., using a `frozenset` for fast $O(1)$ lookup) before injecting them into queries.
