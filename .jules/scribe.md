## 2025-05-11 - Documenting Pipeline Entry Points **Drift Found:** The main execution pipelines (`PromptTranspilerPipeline.run` and `transpile_pipeline`) were missing `Returns` and `Raises` sections, and `transpile_pipeline` completely lacked argument documentation despite being the primary entry point for the CLI/API. **Learning:** As entry points evolve to accept more configuration overrides (e.g. `score_threshold`, `scoring_algo`), their docstrings often lag behind the new type hints, leaving callers guessing about expected exceptions or return payloads. **Action:** Always add `Returns` and `Raises` blocks to public pipeline orchestrators so consumers know exactly what models to handle and which runtime errors to catch.
